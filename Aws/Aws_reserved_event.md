# EC2 예약된 유지관리 이벤트

먼저 예약된 유지관리 이벤트란? 
AWS는 클라우드 서비스지만 실질적으로 서울 리전 내 가용영역의 데이터 센터에서는 사용중인 리소스가 호스팅 되는 서버가 존재한다.

하지만 해당 서버또한 하드웨어 이기 때문에 지속적인 고나리가 필요하다. 그렇기 때문에 해당 하드웨어 장비를 교체해주기 위해서 실행되는것이 유지관리 이벤트 이다.

유지관리 이벤트가 실행되면 해당 인스턴스의 호스트는 다른 호스트로 변경된다 해당 실행 과정중에는 필수적으로 재부팅이 일어나며 해당 재부팅시 탄력적 IP 를 적용하지 않았다면 해당 퍼블릭 IP 는 변경됩니다. 

예정된 유지보수 기간 전에 영향을 받는 인스턴스의 상태를 중지 후 다시 시작으로 변경하게 된다면, 예약된 유지관리의 상태는 complete로 변경됩니다.
